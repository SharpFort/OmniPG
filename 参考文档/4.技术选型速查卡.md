## 📐 model.conf 模板（标准 RBAC）

```ini
[request_definition]
r = sub, obj, act

[policy_definition]
p = sub, obj, act

[role_definition]
g = _, _

[policy_effect]
e = some(where (p.eft == allow))

[matchers]
m = g(r.sub, p.sub) && keyMatch2(r.obj, p.obj) && r.act == p.act
```

**关键点：**
- `keyMatch2` 支持 RESTful 路径通配符（如 `/api/users/:id` 匹配 `/api/users/42`）
- `g(r.sub, p.sub)` 实现角色继承（用户通过 g 规则绑定角色，角色通过 p 规则绑定权限）

## ⚡ 数据流向速查

### 写入路径：管理员修改权限
```
Admin UI → POST /sys_user_role → PostgREST → INSERT INTO sys_user_role
                                                      │
                                          casbin_rule 视图自动反映变更
                                                      │
                                          pg_notify('casbin_channel')
                                                      │
                                          APISIX 收到通知 → 重新 LoadPolicy()
```

### 读取路径：用户访问 API
```
Client → GET /api/users/42 (Header: Authorization Bearer <JWT>)
   │
   ▼
APISIX:
  1. jwt-auth 插件解析 JWT → sub = "alice"
  2. 提取 obj = "/api/users/42", act = "GET"
  3. authz-casbin 调用 Enforce("alice", "/api/users/42", "GET")
  4. 内存匹配: g("alice", "viewer") && keyMatch2("/api/users/42", "/api/users/:id")
  5. ✅ 通过 → 转发到 PostgREST
  或   ❌ 拒绝 → 返回 403
```

## 🔧 关键配置速查

### PostgREST 环境变量
```bash
PGRST_DB_URI=postgres://app_user:password@localhost:5432/app_db
PGRST_DB_SCHEMA=api           # 暴露的 schema
PGRST_DB_ANON_ROLE=anon       # 匿名用户角色
PGRST_JWT_SECRET=your_secret  # 与 pgjwt 使用的密钥一致
PGRST_OPENAPI_SERVER_PROXY_URI=http://localhost:3000
```

### APISIX 路由配置（核心）
```yaml
routes:
  - uri: /api/*
    plugins:
      jwt-auth: {}            # JWT 校验
      authz-casbin:           # Casbin 鉴权
        model: |
          [request_definition]
          r = sub, obj, act
          [policy_definition]
          p = sub, obj, act
          [role_definition]
          g = _, _
          [policy_effect]
          e = some(where (p.eft == allow))
          [matchers]
          m = g(r.sub, p.sub) && keyMatch2(r.obj, p.obj) && r.act == p.act
        # 数据库适配器配置（待确认）
      limit-req:              # 限流
        rate: 100
        burst: 50
    upstream:
      nodes:
        "postgrest:3000": 1
```

## ⚠️ 关键注意事项 Top 10

| # | 注意事项 | 严重性 |
|:---:|:---|:---:|
| 1 | **APISIX 监听 PG NOTIFY 的具体方案必须先行验证** | 🔴 阻塞 |
| 2 | 不要使用 PostgreSQL `INHERITS` 表继承共享审计字段 | 🟡 重要 |
| 3 | `p` 规则和 `g` 规则必须独立，避免笛卡尔积爆炸 | 🟡 重要 |
| 4 | 严禁把高 CPU 计算放入 PL/pgSQL（外部 HTTP/图片处理等） | 🟡 重要 |
| 5 | JWT Secret 必须安全存储，多环境隔离，支持轮转 | 🔴 阻塞 |
| 6 | `pg_timetable` 是 AGPL 许可，生产使用需评估合规 | 🟢 增强 |
| 7 | 强制使用外键 CASCADE，确保删除角色时权限自动清除 | 🟡 重要 |
| 8 | casbin_rule 视图不要通过 PostgREST 暴露写接口 | 🟡 重要 |
| 9 | PL/pgSQL 函数应优先用 SECURITY INVOKER（最小权限） | 🟢 增强 |
| 10 | 定期做全量策略重载（对账），防止缓存不一致 | 🟡 重要 |
