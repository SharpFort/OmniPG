## ⚠️ 整合时需注意的 3 个小事项

虽然所有代码片段都已就绪，但在拼接为完整系统时需要注意：

### 1. refresh_token_rtr() 需更新 JWT payload

第二轮给出的 `refresh_token_rtr()` 函数中，生成新 JWT 的代码使用的是旧的 payload 结构（只有 role、username、exp）。需要参照第四轮 `user_login_sso` 的新 payload 结构同步更新——加入 `jti`、`user_id`、`tenant_id`、`dept_id`、`roles` 数组，并使用 `generate_rs256_jwt()` 签名。

### 2. sys_user_session 表需要新增 active_jti 列

第四轮疑点 3 的 `user_login_sso` 和疑点 5 的 `blacklist_at_on_role_change` 都依赖 `sys_user_session.active_jti` 字段，但第二轮给出的原始 `sys_user_session` DDL 中没有这个字段。

```sql
ALTER TABLE sys_user_session ADD COLUMN active_jti VARCHAR(50);
```

### 3. 数据库 Schema 命名统一

不同轮次的代码中 Schema 名称略有差异（`api_v1` vs `public` vs `sys_schema`），整合时需要统一。
