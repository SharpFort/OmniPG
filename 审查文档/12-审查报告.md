### P1-2. Phase 2 "Create User" 测试密码字段错误 — 创建后无法登录

> **位置：** §2.1 第173行

```powershell
$body = '{"username":"testuser_$(Get-Random)","password_hash":"test","tenant_id":"tenant_default"}'
```

**问题：**
与 11 审查 P1-1 相同的问题：
- `password_hash` 字段接收明文 `"test"`
- 07 后端 `user_login_sso()` 使用 `crypt(p_password, v_user.password_hash)` 验证
- `crypt("test", "test")` ≠ `"test"` → 登录永远失败

**后果：**
- Phase 2 "Create User" 测试本身会通过（HTTP 200）
- 但后续如果测试"用新用户登录"会失败
- 测试数据污染数据库（存在无法登录的僵尸用户）

**修复：**
```powershell
# 方案 A：使用 bcrypt 哈希（需前端或脚本预计算）
$hashedPassword = "$2b$10$..."  # 预计算的 bcrypt("test", 10)
$body = "{\"username\":\"testuser_$(Get-Random)\",\"password_hash\":\"$hashedPassword\",\"tenant_id\":\"tenant_default\"}"

# 方案 B（推荐）：调用存储过程或 RPC 创建用户
$body = '{"p_username":"testuser","p_password":"test","p_tenant_id":"tenant_default"}'
$resp = Invoke-RestMethod -Uri "$BaseUrl/api/v1/rpc/create_user" -Method POST -Body $body
```

### P1-3. Phase 4 "Policy Sync" 测试逻辑永真 — 永远 PASS

> **位置：** §2.1 第260-265行

```powershell
Run-Test "REALTIME" "Policy Sync" "策略同步链路正常" {
    $logs = docker logs policy-syncer --tail=5 2>$null
    $logs | Select-String "Successfully synchronized|listening|leader"
    $true   # ← ⚠️  永远返回 $true
}
```

**问题：**
- `Select-String` 返回 `MatchInfo` 对象或 `$null`
- 无论是否匹配，脚本继续执行 `$true`
- `Run-Test` 函数检查 `$result` 是否为真 → 永远通过

**修复：**
```powershell
Run-Test "REALTIME" "Policy Sync" "策略同步链路正常" {
    $logs = docker logs policy-syncer --tail=20 2>$null
    $matched = $logs | Select-String "Successfully synchronized|listening|leader"
    $matched -ne $null -and $matched.Count -gt 0
}
```

### P1-4. 缺少测试数据清理 — 多次运行后数据库污染

> **位置：** §2.1 Phase 2 创建测试数据后无清理

**问题：**
- Phase 2 创建 `testuser_*` 用户和 `test_role` 角色
- 多次运行脚本会导致：
  - 同名用户名冲突（`ON CONFLICT DO NOTHING` 跳过但测试可能误判为失败）
  - 测试角色累积
  - `sys_role_api` / `sys_role_menu` 关联表垃圾数据

**修复：**
在脚本末尾添加 Phase 8 清理：
```powershell
Write-Host ""
Write-Host "📋 Phase 8: 清理测试数据" -ForegroundColor Yellow

# 删除测试用户
Invoke-RestMethod -Uri "$BaseUrl/api/v1/sys_user?username=like.testuser_*" -Method DELETE -Headers $headers
# 删除测试角色
Invoke-RestMethod -Uri "$BaseUrl/api/v1/sys_role?role_code=eq.test_role" -Method DELETE -Headers $headers
```

### P2-2. 多租户隔离测试不充分 — 未真正验证 RLS

> **位置：** §2.1 Phase 5

```powershell
Run-Test "TENANT" "Cross-Tenant Blocked" "跨租户访问被阻止" {
    $resp = Invoke-RestMethod -Uri "$BaseUrl/api/v1/sys_user?tenant_id=eq.tenant_nonexistent&limit=1" -Headers $headers
    $resp.Count -eq 0
}
```

**问题：**
- 这个测试查询 `tenant_id=eq.tenant_nonexistent`，返回空数组
- 但这**不能证明 RLS 生效**，只是证明没有这个租户的数据
- 真正的 RLS 测试需要：用租户 A 的 JWT 查询租户 B 的数据

**修复：**
```powershell
# 1. 创建租户 B 的用户
# 2. 用 admin（租户 A）的 JWT 查询租户 B 的用户
# 3. 验证返回空数组（RLS 过滤）

Run-Test "TENANT" "RLS Isolation" "RLS 隔离跨租户数据" {
    # 创建租户 B
    $tenantBBody = '{"id":"tenant_b_test","dept_name":"Test Tenant B"}'
    Invoke-RestMethod -Uri "$BaseUrl/api/v1/sys_department" -Method POST -Headers $headers -Body $tenantBBody
    
    # 用 admin JWT 查询租户 B 的用户 → 应该为空
    $resp = Invoke-RestMethod -Uri "$BaseUrl/api/v1/sys_user?tenant_id=eq.tenant_b_test" -Headers $headers
    $resp.Count -eq 0
}
```

### P2-3. 缺少 pg_notify 实时同步端到端测试

> **位置：** §2.1 Phase 6 仅检查触发器存在

**问题：**
Phase 6 的 "pg_notify Trigger" 测试只检查触发器是否存在，没有真正测试：
1. INSERT sys_role_api → pg_notify 发送
2. Syncer 收到通知 → 执行 Sync
3. APISIX etcd 中策略更新

**修复：**
```powershell
Run-Test "SYNC" "Real-time Sync" "pg_notify 实时同步" {
    # 1. 记录当前 APISIX 策略哈希
    $before = Invoke-RestMethod -Uri "$BaseUrl/api/v1/casbin_rule?limit=1" -Headers $headers
    
    # 2. 插入新的角色-API 关联
    $newApiBody = '{"path":"/api/v1/test_sync","method":"GET","api_name":"Sync Test"}'
    $apiResp = Invoke-RestMethod -Uri "$BaseUrl/api/v1/sys_api" -Method POST -Headers $headers -Body $newApiBody
    $apiId = $apiResp[0].id
    
    $roleApiBody = "{\"role_id\":\"$testRoleId\",\"api_id\":\"$apiId\"}"
    Invoke-RestMethod -Uri "$BaseUrl/api/v1/sys_role_api" -Method POST -Headers $headers -Body $roleApiBody | Out-Null

    Start-Sleep -Seconds 3

    $after.Count -gt 0
```

### P2-4. 性能基线测试无重试机制 — 首次运行可能超时

> **位置：** §4.2 API 鉴权延迟

**问题：**
- 脚本注释 `# Warm up` 但只执行 1 次
- Docker 容器冷启动时首次请求可能 >100ms
- 没有排除异常值（outlier）

**修复：**
```powershell
# Warm up（5 次）
1..5 | ForEach-Object {
    Invoke-RestMethod -Uri $uri -Headers $headers | Out-Null
}

# 排除前 5 次（JIT 编译 + 连接池预热）
$times = @()
for ($i = 0; $i -lt 105; $i++) {
    $sw = [System.Diagnostics.Stopwatch]::StartNew()
    Invoke-RestMethod -Uri $uri -Headers $headers | Out-Null
    $sw.Stop()
    if ($i -ge 5) { $times += $sw.ElapsedMilliseconds }
}
```
