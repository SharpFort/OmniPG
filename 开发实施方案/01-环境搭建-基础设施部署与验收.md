## 3. Docker Compose 编排

### 3.1 创建 docker-compose.yml

> **v5.1 设计说明：**
> - ❌ 删除 `redis` 容器（生产由 Pigsty 管理，开发可后续按需添加）
> - ❌ 删除 `minio` 容器（生产由 Pigsty MinIO 内置模板管理）
> - ❌ 删除 `loki` 容器（生产由 Pigsty VictoriaLogs 替代）
> - ❌ **删除 `mysql` 容器**（Casdoor 原生支持 PostgreSQL，无需额外 MySQL）
> - ✅ 新增 `pgbouncer` 容器（对齐生产连接池行为）
> - ✅ 新增 `casdoor` 容器（OAuth/IAM 认证，**直连 PostgreSQL**）
> - ✅ 所有数据由 **单一 PostgreSQL** 实例管理：`app_db` + `casdoor` 两个 database

在 `D:\WeChat Files\xiangmu\源码\` 下创建 `docker-compose.yml`：

```yaml
version: '3.8'

# ==============================================================================
# 网络定义
# ==============================================================================
networks:
  app-net:
    driver: bridge
    ipam:
      config:
        - subnet: 172.20.0.0/16

# ==============================================================================
# 数据卷定义
# ==============================================================================
volumes:
  pg_data:
  etcd_data:
  casdoor_data:

# ==============================================================================
# 服务定义
# ==============================================================================
services:

  # ============================================================================
  # 1. PostgreSQL 数据库（主数据库：app_db + casdoor）
  # ============================================================================
  # 注意：此容器仅用于开发环境功能验证
  # 生产环境使用 Pigsty 部署 PostgreSQL + Patroni HA + Pgbouncer
  # ============================================================================
  postgres:
    image: pgcharles/pgtap:18
    container_name: app-postgres
    restart: unless-stopped
    environment:
      POSTGRES_DB: app_db
      POSTGRES_USER: app_owner
      POSTGRES_PASSWORD: ${DB_PASSWORD:-dev_password_change_me}
    ports:
      - "5432:5432"
    volumes:
      - pg_data:/var/lib/postgresql/data
      - ./db/init:/docker-entrypoint-initdb.d  # 初始化脚本
    networks:
      - app-net
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U app_owner -d app_db"]
      interval: 10s
      timeout: 5s
      retries: 5
    command: >
      -c shared_preload_libraries='pgaudit,pg_cron,pgsodium,pg_net,pg_graphql'
      -c cron.database_name='app_db'
      -c pg_net.database_name='app_db'

  # ============================================================================
  # 2. Pgbouncer 连接池（开发环境补充）
  # ============================================================================
  # 生产环境此路由由 Pigsty 内置 Pgbouncer 替代（Port 5433/5434）
  # ============================================================================
  pgbouncer:
    image: pgbouncer/pgbouncer:1.24
    container_name: app-pgbouncer
    restart: unless-stopped
    ports:
      - "5433:6432"
    volumes:
      - ./pgbouncer/pgbouncer.ini:/etc/pgbouncer/pgbouncer.ini:ro
      - ./pgbouncer/userlist.txt:/etc/pgbouncer/userlist.txt:ro
    networks:
      - app-net
    depends_on:
      postgres:
        condition: service_healthy

  # ============================================================================
  # 3. etcd (APISIX 配置存储)
  # ============================================================================
  # 开发环境单节点，生产环境使用 Pigsty etcd ×3
  # Pigsty 的 etcd 和 APISIX 的 etcd 共用同一集群，/service/ 和 /apisix/ 前缀隔离
  # ============================================================================
  etcd:
    image: bitnami/etcd:3.6
    container_name: app-etcd
    restart: unless-stopped
    environment:
      ETCD_NAME: etcd-dev-node
      ALLOW_NONE_AUTHENTICATION: "yes"
      ETCD_ADVERTISE_CLIENT_URLS: http://0.0.0.0:2379
      ETCD_LISTEN_CLIENT_URLS: http://0.0.0.0:2379
      ETCD_DATA_DIR: /bitnami/etcd/data
    ports:
      - "2379:2379"
    volumes:
      - etcd_data:/bitnami/etcd/data
    networks:
      - app-net
    healthcheck:
      test: ["CMD", "etcdctl", "endpoint", "health"]
      interval: 10s
      timeout: 5s
      retries: 5

  # ============================================================================
  # 4. APISIX API 网关
  # ============================================================================
  apisix:
    image: apache/apisix:3.17.0-debian
    container_name: app-apisix
    restart: unless-stopped
    environment:
      APISIX_STAND_ALONE: "false"
    ports:
      - "9080:9080"
      - "9443:9443"
      - "9180:9180"
    volumes:
      - ./apisix/config.yaml:/usr/local/apisix/conf/config.yaml:ro
      - ./apisix/apisix.yaml:/usr/local/apisix/conf/apisix.yaml:ro
    networks:
      - app-net
    depends_on:
      etcd:
        condition: service_healthy
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:9080/apisix/status"]
      interval: 10s
      timeout: 5s
      retries: 5

  # ============================================================================
  # 5. PostgREST API 自动生成引擎
  # ============================================================================
  postgrest:
    image: postgrest/postgrest:v14.14
    container_name: app-postgrest
    restart: unless-stopped
    environment:
      PGRST_DB_URI: postgres://authenticator:authenticator_dev_pass@pgbouncer:6432/app_db?sslmode=disable
      PGRST_DB_SCHEMAS: api_v1
      PGRST_DB_ANON_ROLE: web_anon
      PGRST_JWT_SECRET: ${JWT_SECRET}
      PGRST_DB_PRE_REQUEST: api_v1.check_token_blacklist
      PGRST_OPENAPI_SERVER_PROXY_URI: http://localhost:3000
      PGRST_SERVER_HOST: 0.0.0.0
      PGRST_SERVER_PORT: "3000"
    ports:
      - "3000:3000"
    networks:
      - app-net
    depends_on:
      pgbouncer:
        condition: service_started

  # ============================================================================
  # 6. Casdoor IAM/OAuth 认证服务（直连 PostgreSQL，无需 MySQL）
  # ============================================================================
  # 使用 PostgreSQL 作为后端数据库，数据存储于 casdoor database
  # ============================================================================
  casdoor:
    image: casbin/casdoor:latest
    container_name: app-casdoor
    restart: unless-stopped
    ports:
      - "8000:8000"
    environment:
      driverName: postgres
      dataSourceName: "user=casdoor password=${CASDOOR_DB_PASSWORD:-casdoor_dev_pass} host=pgbouncer port=6432 sslmode=disable dbname=casdoor"
      runMode: dev
      httpPort: 8000
    volumes:
      - casdoor_data:/app/conf
    networks:
      - app-net
    depends_on:
      pgbouncer:
        condition: service_started
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/api/health"]
      interval: 10s
      timeout: 5s
      retries: 5

  # ============================================================================
  # 7. Swagger UI API 文档
  # ============================================================================
  swagger-ui:
    image: swaggerapi/swagger-ui:v5.18.2
    container_name: app-swagger
    restart: unless-stopped
    environment:
      API_URL: http://localhost:3000/
    ports:
      - "8080:8080"
    networks:
      - app-net
    depends_on:
      - postgrest
```

### 4.2 创建初始 Schema 和角色

创建 `D:\WeChat Files\xiangmu\源码\db\init\02-schemas.sql`：

```sql
-- 创建业务 Schema
CREATE SCHEMA IF NOT EXISTS api_v1;
COMMENT ON SCHEMA api_v1 IS 'PostgREST 暴露的业务 API Schema';

-- 创建匿名角色（无权访问数据）
DO $$
BEGIN
    IF NOT EXISTS (SELECT FROM pg_roles WHERE rolname = 'web_anon') THEN
        CREATE ROLE web_anon NOLOGIN NOINHERIT;
    END IF;
END
$$;

-- 创建认证用户角色
DO $$
BEGIN
    IF NOT EXISTS (SELECT FROM pg_roles WHERE rolname = 'authenticated') THEN
        CREATE ROLE authenticated NOLOGIN NOINHERIT;
    END IF;
END
$$;

-- 创建 authenticator 角色（PostgREST 连接用的 LOGIN 角色）
DO $$
BEGIN
    IF NOT EXISTS (SELECT FROM pg_roles WHERE rolname = 'authenticator') THEN
        CREATE ROLE authenticator LOGIN NOINHERIT PASSWORD 'authenticator_dev_pass';
    END IF;
END
$$;

-- 建立角色切换链：authenticator → web_anon / authenticated
GRANT web_anon TO authenticator;
GRANT authenticated TO authenticator;

-- 授予 Schema 使用权
GRANT USAGE ON SCHEMA api_v1 TO web_anon;
GRANT USAGE ON SCHEMA api_v1 TO authenticated;

-- web_anon 默认无任何表权限（安全第一）
-- authenticated 的权限在后续 migration 中根据表逐步授予
```

### 4.3 创建 Casdoor 数据库和用户

> 由于 `docker-entrypoint-initdb.d` 默认在 `POSTGRES_DB`（app_db）中执行，以下脚本需要**在容器首次启动后手动执行**。

创建 `D:\WeChat Files\xiangmu\源码\db\init\03-casdoor-db.sql`：

```sql
-- ============================================================================
-- Casdoor 数据库初始化脚本
-- 使用方法：docker exec -it app-postgres psql -U postgres -f /path/to/03-casdoor-db.sql
-- ============================================================================

-- 1. 创建 casdoor 用户
DO $$
BEGIN
    IF NOT EXISTS (SELECT FROM pg_roles WHERE rolname = 'casdoor') THEN
        CREATE ROLE casdoor LOGIN PASSWORD 'casdoor_dev_pass';
    END IF;
END
$$;

-- 2. 创建 casdoor 数据库
SELECT 'CREATE DATABASE casdoor OWNER casdoor'
WHERE NOT EXISTS (SELECT FROM pg_database WHERE datname = 'casdoor')\gexec

-- 3. 授予权限
GRANT ALL PRIVILEGES ON DATABASE casdoor TO casdoor;

-- 4. 连接到 casdoor 数据库，启用必要扩展
\connect casdoor

CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
CREATE EXTENSION IF NOT EXISTS "pgcrypto";

-- 5. 确认创建成功
SELECT current_database(), current_user;
```

## 11. 常见问题排查

| 问题 | 解决方案 |
|:---|:---|
| 端口冲突 | `netstat -ano \| findstr :5432` 查看占用，修改 docker-compose.yml 端口映射 |
| PG 启动失败 | 检查 `pg_data` 卷权限，删除卷重建：`docker compose down -v && docker compose up -d` |
| etcd 内存不足 | 减少 etcd 内存限制或增加宿主机内存 |
| Pgbouncer 连接失败 | 确认 `pgbouncer/userlist.txt` 中的密码与 PG 用户密码匹配 |
| Casdoor 无法启动 | 检查 PostgreSQL 容器是否已健康 + casdoor database 是否已创建 |
| PostgREST 连接 Pgbouncer 失败 | 确认 Pgbouncer 容器已启动且 `pool_mode = session` |
| Dbmate 连接失败 | 确认 PostgreSQL 容器已健康启动（等 10-15 秒重试） |
| Windows 路径问题 | 所有路径使用正斜杠 `/` 或双反斜杠 `\\` |
| pg_graphql 扩展不存在 | 进入容器手动安装：`docker exec -it app-postgres psql -U app_owner -d app_db -c "CREATE EXTENSION pg_graphql;"` |
| Casdoor JDBC URL 格式错误 | 格式：`user=xxx password=xxx host=xxx port=xxx sslmode=disable dbname=casdoor` |

## 12. etcd 双集群说明

### 12.1 架构决策

本方案采用 **单集群共用** 模式，通过 Key 前缀隔离：

```
单一 etcd 集群（开发环境单节点，生产环境 ×3）
├── /service/ → Patroni DCS Leader Election（生产环境）
├── /apisix/  → APISIX 路由配置 + Casbin Policy
└── (开发环境仅使用 /apisix/ 前缀)
```

### 12.2 权限隔离（生产环境）

```bash
# 通过 etcd RBAC 隔离 APISIX 和 Patroni 的访问
etcdctl role add apisix-role
etcdctl role grant-permission apisix-role --prefix /apisix/ readwrite
etcdctl user add apisix:your-password
etcdctl user grant-role apisix apisix-role
```

### 12.3 开发环境配置

开发环境使用单节点 etcd，不启用权限控制（`ALLOW_NONE_AUTHENTICATION: "yes"`），APISIX 仅使用 `/apisix/` 前缀。

## 13. Casdoor + PostgreSQL 集成说明

### 13.1 为什么不需要 MySQL

Casdoor 官方文档明确列出支持的数据库驱动：

| 驱动 | 说明 |
|:---|:---|
| `postgres` | PostgreSQL（推荐生产） |
| `mysql` | MySQL |
| `sqlite` | SQLite（推荐开发快速启动） |

结论：**PostgreSQL 是 Casdoor 的一级公民，完全不需要额外引入 MySQL。**

### 13.2 Casdoor ↗ PG 连接配置

```yaml
# docker-compose.yml 中的 casdoor 环境变量
environment:
  driverName: postgres
  dataSourceName: "user=casdoor password=${CASDOOR_DB_PASSWORD} host=pgbouncer port=6432 sslmode=disable dbname=casdoor"
```

### 13.3 数据库规划

| Database | 用途 | 属主 |
|:---|:---|:---|
| `app_db` | 11张 RBAC 业务表 + 视图 + 函数 | `app_owner` |
| `casdoor` | Casdoor 用户/应用/组织配置 | `casdoor` |
| `postgres` | PostgreSQL 系统数据库 | `postgres` |

三个 database 在单一 PG 实例中，开发和生产环境完全一致。
