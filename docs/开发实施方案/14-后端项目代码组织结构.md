## 设计要点

### Schema 级物理隔离

| Schema | 用途 | 暴露方式 |
|:---|:---|:---|
| `public` | 存放 11 张 `sys_*` RBAC 表、会话表、黑名单表 | 不直接暴露，通过 `api_v1` 视图访问 |
| `sales` | 存放订单、订单明细等销售业务表 | 不直接暴露，通过 `api_v1` 视图访问 |
| `inventory` | 存放库存、仓库等库存业务表 | 不直接暴露，通过 `api_v1` 视图访问 |
| `api_v1` | PostgREST **唯一暴露给网关的 Schema**，只放视图和 RPC 函数 | 直接暴露给前端/网关 |

### DDL 与 src 的边界

| 类型 | 存放位置 | 管理方式 |
|:---|:---|:---|
| 物理表 DDL、索引、种子数据 | `db/migrations/` 时间戳文件 | Dbmate 增量迁移，**只增不改** |
| 视图、函数、触发器 | `db/src/<schema>/` 独立 `.sql` 文件 | `CREATE OR REPLACE` 幂等源码 |

### Makefile 关键目标

```makefile
dev-env         # 启动本地开发环境 (docker compose up -d)
db-migrate      # 执行数据库向前迁移 (dbmate up)
db-rollback     # 回滚最近一次迁移 (dbmate rollback --single)
apply-src       # 将 db/src 中的幂等源码刷入数据库
test-db         # 运行 pgTAP 单元测试
test-syncer     # 运行 Go 同步器单元测试
test-e2e        # 运行全链路 E2E 集成测试
test-all        # 一键执行全栈测试
run-syncer      # 启动 Policy Syncer
```
