# 13 — 生产部署方案

> **定位：** 提供完整的生产环境部署方案，涵盖 Kubernetes/Swarm 编排、安全加固、监控告警、备份恢复和 CI/CD。Agent 按本文档可将开发环境安全、高效地迁移到生产。
> **前置依赖：** 08-Docker Compose、12-端到端集成测试（验收通过）
> **产出物：** 生产级编排配置 + 安全清单 + 运维脚本 + 监控体系
> **预计耗时：** 4-8 小时（含问题排查）

- 3 个 Worker 节点（4C/8G 起步）
- 1 个 Master 节点（2C/4G）
- 1 个 Ingress 节点（2C/2G，可选）

### 2.1 网络安全

```yaml
# Kubernetes NetworkPolicy 示例
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: deny-all
  namespace: system
spec:
  podSelector: {}
  policyTypes:
    - Ingress
    - Egress

---
# [增强 P2-1] 只允许 Ingress 访问 APISIX HTTP (9080)
# APISIX Admin API (9180) 仅通过 apisix-admin 服务在集群内部暴露
# ⚠️ 禁止将 apisix-admin 暴露到 Ingress / 公网
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-ingress-to-apisix
  namespace: system
spec:
  podSelector:
    matchLabels:
      app: apisix
  ingress:
    - from:
        - namespaceSelector:
            matchLabels:
              name: ingress-nginx
      ports:
        - port: 9080
```

```yaml
            - log_min_duration_statement=1000

              cpu: "4"
              memory: "8Gi"

              mountPath: /var/lib/postgresql/data

            exec:
              command: ["pg_isready", "-U", "postgres"]

            periodSeconds: 10
          livenessProbe:
            exec:
              command: ["pg_isready", "-U", "postgres"]
            initialDelaySeconds: 30
            periodSeconds: 30
  volumeClaimTemplates:
```

## 6. 备份与恢复

### 6.1 PostgreSQL 备份策略

```powershell
# 每天凌晨全量备份 + WAL 归档
$backupScript = @'
#!/bin/bash
BACKUP_DIR="/backup/postgres"
DATE=$(date +%Y%m%d_%H%M%S)
RETENTION_DAYS=30

# 全量备份 (pg_basebackup)
pg_basebackup -D $BACKUP_DIR/base/$DATE -Ft -z -P

# 或 pg_dump 逻辑备份
pg_dump -Fc app_db > $BACKUP_DIR/dump_$DATE.dump

# 上传至 S3
aws s3 sync $BACKUP_DIR s3://my-backup-bucket/postgres/

# 清理旧备份（保留30天）
find $BACKUP_DIR -name "base_*" -mtime +30 -delete
find $BACKUP_DIR -name "dump_*" -mtime +30 -delete
'@

# 部署为 CronJob
kubectl apply -f - <<EOF
apiVersion: batch/v1
kind: CronJob
metadata:
  name: pg-backup
  namespace: system
spec:
  schedule: "0 2 * * *"  # 每天凌晨 2 点
  jobTemplate:
    spec:
      template:
        spec:
          containers:
            - name: backup
              image: postgres:18-alpine
              env:
                - name: PGPASSWORD
                  valueFrom:
                    secretKeyRef:
                      name: app-secrets
                      key: db-password
                - name: AWS_ACCESS_KEY_ID
                  valueFrom:
                    secretKeyRef:
                      name: aws-creds
                      key: access-key
                - name: AWS_SECRET_ACCESS_KEY
                  valueFrom:
                    secretKeyRef:
                      name: aws-creds
                      key: secret-key
                - name: AWS_REGION
                  value: "us-east-1"  # [增强 P2-3] 备份目标区域
                - /bin/sh
                - -c

 \
                  aws s3 cp - s3://my-backup-bucket/postgres/$(date +%Y%m%d).dump
          restartPolicy: Never
  successfulJobsHistoryLimit: 3
  failedJobsHistoryLimit: 1
EOF
```

## 7. CI/CD 流水线

### 7.1 GitHub Actions 示例

```yaml
# .github/workflows/production.yml
name: Production Deploy

on:
  push:
    branches: [main]

jobs:
  build-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Build Backend

          docker build -t $REGISTRY/postgrest:${{ github.sha }} -f postgrest/Dockerfile ./postgrest/
          docker build -t $REGISTRY/syncer:${{ github.sha }} -f syncer/Dockerfile ./syncer/

      - name: Build Frontend

          cd frontend/admin-ui
          npm ci
          npm run build
          docker build -t $REGISTRY/frontend:${{ github.sha }} .

      - name: Security Trivy Scan
        uses: aquasecurity/trivy-action@master
        with:
          image-ref: $REGISTRY/syncer:${{ github.sha }}
          format: table
          exit-code: '1'
          severity: 'CRITICAL,HIGH'

  deploy:
    needs: build-test
    runs-on: ubuntu-latest
    environment: production
    steps:
      - uses: actions/checkout@v4

      - name: Push Images

          docker login $REGISTRY -u $REGISTRY_USER -p $REGISTRY_PASS
          docker push $REGISTRY/postgrest:${{ github.sha }}
          docker push $REGISTRY/syncer:${{ github.sha }}
          docker push $REGISTRY/frontend:${{ github.sha }}

      - name: Deploy to K8s
        uses: azure/k8s-deploy@v1
        with:
          namespace: system

            k8s/postgrest-deployment.yaml
```
