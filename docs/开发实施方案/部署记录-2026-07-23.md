## 二、执行步骤

### 2.1 清理手动安装

最初手动通过 apt 安装了 PG 18、pgBouncer、Redis。为统一由 Pigsty 管理，全部卸载：

```bash
sudo apt remove -y postgresql-18 postgresql-client-18 postgresql-contrib-18 \
  postgresql-18-jit postgresql-common pgbouncer redis-server redis-tools \
  postgresql-client-common
sudo rm -rf /var/lib/postgresql/18 /etc/postgresql/18 /etc/pgbouncer \
  /etc/redis /var/lib/redis
```

### 2.2 Pigsty 部署

```bash
# 下载
curl -fsSL https://pigsty.cc/get | bash -s v4.4.0

cd ~/pigsty

# 生成配置
./configure -i 192.168.0.128 -n -s

# 自定义 pigsty.yml（启用 INFRA/PGSQL/REDIS/ETCD 模块）
# 见 deploy/pigsty.yml

# 部署（分步执行避免超时）
./infra.yml    # INFRA 模块
./etcd.yml     # etcd 模块（注意：组名需匹配 etcd_cluster 变量）
./pgsql.yml    # PGSQL 模块（仅 pgcrypto 扩展报错，PG18 已内置）
./redis.yml    # REDIS 模块（需要 redis_instances 变量）
```

### 2.3 数据库初始化

```bash
# 创建用户
su - postgres -c "psql -c \"CREATE USER app_owner WITH PASSWORD 'dev_password_change_me' CREATEDB;\""
su - postgres -c "psql -c \"CREATE USER authenticator WITH PASSWORD 'authenticator_dev_pass';\""
su - postgres -c "psql -c \"CREATE USER casdoor WITH PASSWORD 'casdoor_dev_pass';\""
su - postgres -c "psql -c \"CREATE ROLE web_anon NOLOGIN;\""

# 创建数据库
su - postgres -c "psql -c \"CREATE DATABASE app_db OWNER app_owner;\""
su - postgres -c "psql -c \"CREATE DATABASE casdoor OWNER casdoor;\""

# 安装扩展
su - postgres -c "psql -d app_db -c \"CREATE EXTENSION IF NOT EXISTS pgcrypto, pgsodium, pgaudit, pgtap, pg_graphql, pg_cron;\""

# 配置 shared_preload_libraries（pg_net 需要）
echo "shared_preload_libraries = 'pg_net,pg_cron'" >> /etc/postgresql/18/main/postgresql.conf
su - postgres -c "psql -d app_db -c \"CREATE EXTENSION IF NOT EXISTS pg_net;\""

# 重启
systemctl restart postgresql@18-main
```

### 2.4 pgBouncer 配置

Pigsty 已内置 pgBouncer，配置文件 `/etc/pgbouncer/pgbouncer.ini`：

```ini
[databases]
app_db = host=127.0.0.1 port=5432 dbname=app_db
casdoor = host=127.0.0.1 port=5432 dbname=casdoor

[pgbouncer]
listen_addr = 0.0.0.0
listen_port = 6432
auth_type = md5
auth_file = /etc/pgbouncer/userlist.txt
pool_mode = session
max_client_conn = 100
default_pool_size = 20
reserve_pool_size = 5
```

启动命令：`sudo -u postgres /usr/sbin/pgbouncer /etc/pgbouncer/pgbouncer.ini &`

### 2.5 Redis 修复

Pigsty 的 Redis 模块需要 `redis_instances` 变量。手动创建配置后启动：

```bash
# 创建配置目录和文件
mkdir -p /etc/redis /var/lib/redis /var/run/redis /var/log/redis
chown redis:redis /var/lib/redis /var/run/redis /var/log/redis

# 写入配置（见 deploy/redis.conf）
# 启动
systemctl start redis-server
```

### 2.6 Docker Compose 更新

删除了 docker-compose.yml 中已由 Pigsty 原生部署的服务（postgres、pgbouncer、etcd、redis），仅保留无状态服务：

- APISIX (9080)

**注意：** PostgREST 的 3000 端口与 Grafana 冲突，后续需要调整。

## 三、问题与解决方案

| 问题 | 原因 | 解决方案 |
|:---|:---|:---|
| PG TCP 连接超时 | WSL2 DNS 反向解析问题 | 使用 PGPASSWORD 环境变量 |
| PG 重启失败 | sed 替换了注释行 listen_addresses | 修正为正确格式 |
| pgBouncer 连接失败 | HBA 配置 + PG listen_addresses | 添加 md5 HBA + 设置 listen_addresses='localhost' |
| Redis 启动失败 | snap 包冲突 + 配置缺失 | 移除 snap.redis + 手动创建配置 |
| etcd 部署失败 | 组名含连字符导致 Ansible 变量解析失败 | 组名改为 `etcd`（不含连字符） |
| Redis Pigsty 部署失败 | 缺少 `redis_instances` 变量 | 在 pigsty.yml 中添加 `redis_instances: { 6379: {} }` |
| Docker Compose 失败 | 旧版配置包含已部署的服务 | 更新为新版配置，仅保留无状态服务 |
| Syncer 构建超时 | Docker Hub 认证超时 | 网络问题，需重试或使用镜像加速 |
