> **执行模式：** 每次只审查一个文档，用户确认后才继续下一个

---

## 项目背景

本项目旨在构建一套**零后端代码（Database-Driven）的统一权限管理与 API 自动生成系统**，技术栈为：
- **PostgreSQL 18** + **Pigsty v4.3.0**（生产）/ **pgcharles/pgtap:18**（开发）
- **PostgREST v14.14**（RESTful API 自动生成）
- **Apache APISIX 3.17.0**（API 网关 + Casbin 鉴权）
- **Casdoor v3.108.0**（OAuth/IAM 认证，直连 PostgreSQL）
- **Policy Syncer (Go)**（策略实时同步：pg_notify → APISIX etcd）
- **Vue 3 + ART-D Pro**（前端 Admin UI）

**架构核心：** 数据库即代码，业务模型（sys_* 表）通过视图自动映射为 Casbin 运行模型（casbin_rule），所有业务逻辑收敛在 PostgreSQL 内部。

**已审查文档（00-05）：** 项目总纲、环境搭建、数据库建模、API 与认证层、网关与同步器、Casdoor 集成、前端 Admin。

**本次审查范围（06-13）：** Policy Syncer Go 实现、Database Migrations、Docker Compose、PostgREST 配置、APISIX 路由批量配置、前端 API 封装、端到端集成测试、生产部署方案。

## 审查标准

### 技术准确性
- Go 代码是否可编译通过
- SQL 语法是否正确（PostgreSQL 18）
- Docker Compose 配置是否有效
- Kubernetes YAML 是否合法

### 可执行性
- 按文档步骤操作是否能成功
- 环境变量是否完整
- 依赖服务是否就绪

### 一致性
- 与 00-05 已审查文档的架构决策是否一致
- 端口、服务名、版本号是否匹配
- JWT 签发/验证链路是否统一

### 安全性
- 敏感信息是否泄露（密码、密钥）
- 网络安全策略是否完备
- 备份恢复是否可靠
